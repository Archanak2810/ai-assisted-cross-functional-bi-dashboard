# Development Methodology

The dashboard follows an applied design-and-development approach:

1. Define business problems, domain questions, stakeholders, KPIs, and filters.
2. Review schema availability and data readiness.
3. Design Neon PostgreSQL, Python ETL, Streamlit, and reporting architecture.
4. Build reusable data pipeline and domain-level analytics functions.
5. Implement dashboard tabs, filters, chart panels, AI-assisted insights, and export output.
6. Validate database connectivity, row counts, KPI reasonableness, filters, chart rendering, and report download.
7. Deploy through GitHub and Streamlit Cloud.

The methodology diagram is available in `assets/figures/development_methodology.png`.
